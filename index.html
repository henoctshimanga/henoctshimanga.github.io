SmartCourse Finder

From a C++ console application to a full-stack system with modern architecture, database integration, and graph-based logic.

SmartCourse Finder is a course management and advising system designed to help users explore course offerings, manage prerequisites, and generate valid course completion paths. The system allows users to import course data, search courses, view prerequisite relationships, and interact with structured academic planning features. This portfolio demonstrates how a simple data-driven application can evolve into a scalable system by integrating frontend, backend, and database layers.

Algorithms & Data Structures

Graph-Based Logic

Graph-Based Approach

The system uses graph-based data structures to model course relationships. Each course is treated as a node, and prerequisites are represented as directed edges. This approach improves accuracy and scalability compared to a flat lookup model.

  • Flat lookup could not easily support multi-step prerequisite resolution or cycle detection.
  • Graph-based course relationships capture dependencies naturally and support traversal.
  • Prerequisite traversal validates course sequences and supports logical progression.
  • Topological ordering helps identify valid course pathways and detect dependency cycles.

Algorithms in Action

The application implements several key algorithms to ensure correctness and performance:

  • Validate prerequisite relationships: Ensures courses reference valid prerequisites
  • Detect cycles: Identifies circular dependencies that would create impossible sequences
  • Topological sorting: Generates valid course order respecting all prerequisites
  • Graph traversal: Uses depth-first and breadth-first search to explore dependencies

This algorithmic foundation strengthens the system's ability to handle complex prerequisite scenarios and provide reliable course planning recommendations.

Database Design

Persistence & Schema Migration

Migration from H2 to MySQL

The database layer was enhanced by transitioning from an H2 in-memory database to a MySQL database using Spring Data JPA and Hibernate. This progression demonstrates moving from rapid prototyping to production-grade persistence.

  • The initial H2 implementation supported rapid iteration and backend validation.
  • Migration to MySQL introduced durable storage, schema management, and production relevance.
  • JPA/Hibernate persistence maps course entities and prerequisite relationships with auto-generated schema.
  • Resolving Hibernate/MySQL configuration issues reinforced proper persistence setup and compatibility awareness.

Schema Design

The schema reflects a clear separation between structured data storage and application-level logic. While the database stores fundamental records, complex operations are handled in the application layer.

  • Courses table: Stores course_number, course_title, and core course metadata
  • Course prerequisites table: Links courses to their prerequisites using foreign keys
  • Validation for required fields and referential integrity enforced at the database level
  • Cycle detection and dependency validation handled in the application layer using graph algorithms

This design reflects an important trade-off: the database provides structured storage, while the application layer handles advanced problem-solving logic.

Challenges & Learning

Configuration & Integration Challenges

One of the main challenges in this project was configuring the database layer, particularly ensuring compatibility between MySQL and Hibernate. Small configuration issues, such as incorrect dialect settings, prevented the application from starting and required careful troubleshooting.

  • Identified and resolved Hibernate dialect mismatches between development and production databases
  • Debugged JDBC connectivity issues and datasource configuration problems
  • Learned the importance of environment-specific configuration and property management
  • Understood how framework dependencies interact with underlying database systems

Key Takeaways

This experience reinforced critical lessons about system design and integration:

  • Backend, database, and application layers must interact seamlessly for system success
  • Not all complex logic should be handled at the database level; application-level processing is often necessary
  • Configuration management and environment-specific settings are crucial in real-world applications
  • Systematic troubleshooting and attention to error messages are essential development skills

These challenges transformed my understanding of full-stack development from theoretical to practical, reinforcing the importance of careful system design and integration testing.

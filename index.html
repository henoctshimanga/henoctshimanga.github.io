SmartCourse Finder

From a C++ console application to a full-stack system with modern architecture, database integration, and graph-based logic.

SmartCourse Finder is a course management and advising system designed to help users explore course offerings, manage prerequisites, and generate valid course completion paths. The system allows users to import course data, search courses, view prerequisite relationships, and interact with structured academic planning features. This portfolio demonstrates how a simple data-driven application can evolve into a scalable system by integrating frontend, backend, and database layers.

Algorithms & Data Structures

Algorithms and Data Structures

The SmartCourse Finder system was enhanced to improve how course relationships are represented and processed. The original version relied on simple data storage and lookup, which limited its ability to handle complex prerequisite logic.

In the original implementation, courses and prerequisites were treated as flat data. While this allowed basic retrieval, it did not accurately model the relationships between courses, making it difficult to validate dependencies or generate valid course sequences.

To address this limitation, the system was enhanced using graph-based data structures. Each course is represented as a node, and prerequisite relationships are modeled as directed edges between nodes. This structure allows the system to represent dependencies more accurately and efficiently.

  • Validate prerequisite relationships.
  • Detect cycles in course dependencies.
  • Traverse course relationships using depth-first and breadth-first search.
  • Generate a valid course completion order using topological sorting.

This enhancement demonstrates my ability to apply graph data structures to real-world problems, implement traversal algorithms such as DFS and BFS, design logic for dependency resolution, and improve system accuracy and scalability through better data modeling.

This approach introduces additional complexity compared to simple data structures, but it provides significantly more flexibility and accuracy when handling course dependencies. It also shifts part of the system's logic from simple data storage to structured algorithmic processing, which is necessary for solving complex relationship-based problems.

Through this enhancement, I learned how to apply abstract data structures to practical problems. Modeling course relationships as a graph helped me understand how algorithms can be used to enforce rules, validate data, and generate meaningful results in real-world systems.

  • Applied graph data structures to model course and prerequisite relationships.
  • Used traversal algorithms to support dependency validation and relationship exploration.
  • Used topological sorting to produce valid course ordering and identify invalid dependency structures.
  • Balanced added algorithmic complexity against stronger accuracy, scalability, and maintainability.

Database Design

Databases

The SmartCourse Finder system was enhanced by introducing a persistent database layer. The original version relied entirely on in-memory data structures, which limited scalability and data persistence. In the enhanced version, a relational database was integrated using MySQL and Spring Data JPA.

In the original implementation, course data was loaded from a file and stored in memory during runtime. This meant that all data was lost when the application stopped, and the system could not support real-world use cases such as data persistence, updates, or concurrent access.

To address these limitations, the system was enhanced by integrating a MySQL database with Hibernate and Spring Data JPA. This introduced persistent storage and allowed the application to manage course data more effectively. Persistence is handled using JPA repositories, enabling CRUD operations and simplifying database interaction.

  • Courses table: Stores course identifiers, course titles, and core course metadata.
  • Course prerequisites table: Stores prerequisite relationships between courses.
  • JPA repositories: Support create, read, update, and delete operations through structured persistence methods.

While the database stores structured course and prerequisite data, complex logic such as cycle detection and dependency validation is handled in the application layer using graph-based algorithms. This design reflects a deliberate trade-off: the database is used for storage and retrieval, while advanced logic is handled in the application for flexibility and clarity.

The system includes validation rules to maintain consistent and reliable data:

  • Course number and title are required.
  • A course cannot reference itself as a prerequisite.
  • Prerequisites must exist before being assigned.
  • Cycles in course dependencies are prevented.
  • Deletion of courses required by others is restricted.

One of the main challenges in this enhancement was configuring MySQL and Hibernate correctly. Issues related to database connectivity and dialect configuration prevented the application from starting and required careful troubleshooting. This experience highlighted the importance of proper configuration and compatibility between frameworks, as well as the role of persistence layers in full-stack systems.

This enhancement demonstrates my ability to integrate relational databases into full-stack applications, use JPA and Hibernate for persistence, design database schemas, apply validation rules to maintain data integrity, and troubleshoot real-world configuration issues.

Through this enhancement, I gained a deeper understanding of how databases interact with application logic. I learned that designing a system involves balancing responsibilities between the database and the application layer, and that real-world systems require careful configuration, validation, and integration to function reliably.

Configuration & Integration Challenges

One of the main challenges in this project was configuring the database layer, particularly ensuring compatibility between MySQL and Hibernate. Small configuration issues, such as incorrect dialect settings, prevented the application from starting and required careful troubleshooting.

Key Takeaways

This experience reinforced critical lessons about system design and integration:

  • Backend, database, and application layers must interact seamlessly for system success
  • Not all complex logic should be handled at the database level; application-level processing is often necessary
  • Configuration management and environment-specific settings are crucial in real-world applications
  • Systematic troubleshooting and attention to error messages are essential development skills

These challenges transformed my understanding of full-stack development from theoretical to practical, reinforcing the importance of careful system design and integration testing.
